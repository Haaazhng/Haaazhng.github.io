单周期CPU设计

Verilog RISC-V

总述

单周期CPU，即在一个时钟周期内将一条指令执行完后再开始执行下一条指令的CPU。要想设计好单周期CPU，最重要的一点是要深入且准确的理解每条指令在数据通路中的路径。在此前提下，我们将数据通路中的各个模块封装，使用结构化的方式设计CPU，即在CPU得到顶层模块中例化各个模块。数据通路如下所示：

实现了RISC_V 32位指令集架构下的21种指令add、addi、xor、or、and、lui、auipc、lw、sw、jal、jalr、sll、slli、srl、srli、beq、bne、blt、bge、bltu、bgeu
数据通路中有一些用于debug的模块和连线，大家可以自己实现debug模块，也可将这些部分删除，只完成一个CPU。
下面是部分核心模块的具体代码实现。

IMM生成器

传入的参数imm_type指定了需要生成的立即数类型，我们根据这个来处理立即数。一共有六种立即数 ：R，I，S，B，U，J。更准确的说imm_type指示的是执行的指令类型，因为R型指令并不需要立即数，我们把它放在case语句的default语句里默认输出立即数为0，其实这个默认值并不重要，它不会影响指令的执行结果，因为立即数作为输入之一的选择器会选择其他需要的信号。这些选择器其实就是alu_op2_sel和reg_write_sel，即立即数既可能作为ALU操作数的输入（显然的），又可能需要写入目标寄存器（lui）指令。

module Immediate(
    input [31:0] inst,
    input [2:0] imm_type,
    output reg [31:0] imm
    );
    parameter R_type=0, I_type=1, S_type=2,  B_type=3, U_type=4, J_type=5;
    always @(*) begin
        case (imm_type)
            I_type:begin
                imm={{20{inst[31]}}, inst[31:20]};
            end 
            S_type:begin
                imm={{20{inst[31]}}, inst[31:25], inst[11:7]};
            end 
            B_type:begin
                imm={{20{inst[31]}}, inst[7], inst[31:25], inst[11:8], 1'b0};
            end 
            U_type:begin
                imm={{inst[31:12]}, {12{1'b0}}};
            end 
            J_type:begin
                imm={{12{inst[31]}}, inst[19:12], inst[20], inst[30:21], 1'b0};
            end 
            default: begin
                imm=0;
            end
        endcase
    end
endmodule

Control模块

这是至关重要的一个模块，它是整个CPU的控制中枢。前面说过，我们需要理解每一条指令在数据通中的路径，否则，Control模块就无从入手。这个模块设计多个控制信号的产生。为了更加脉络清晰、条理分明的完成这个部分，我制作出了21条指令的对应的控制信号表格：

jalr 0 1 0 1 1 0 1 0 I 0

我们可以观察到，对于大多数控制信号，某个值都占了半数以上。这个值我们可以在每个信号的对应case语句中的default部分默认赋值，甚至用if~else就可以完成一个信号的赋值，这样就减少了代码量。

Branch分支模块

当指令是一个分支指令时，该模块就可以根据控制模块发出的br_type信号处理比较op1和op2，然后输出br跳转信号。br会输入到下一指令地址选择模块中（NPC_SEL）。我们一共有六种跳转指令，其中有两个是无符号比较bltu和bgeu，两个有符号比较blt和bge，而bne和beq无所谓有符号或无符号比较，因为都能输出期望结果。需要注意的是Verilog中的比较逻辑默认是无符号比较，对于blt和bge指令，需要用$signed()来指示操作数其是一个有符号数。经过比较后，若需要跳转，则br=1，否则br=0。

module Branch (
    input [2:0] br_type,
    input [31:0] op1,
    input [31:0] op2,
    output reg br
);
    always @(*) begin
        case (br_type)
            1:begin     //blt
                if($signed(op1) < $signed(op2)) br=1;
                else br=0;
            end 
            2:begin     //beq
                if(op1 == op2) br=1;
                else br=0;
            end
            3:begin     //bne
                if(op1 != op2) br=1;
                else br=0;
            end 
            4:begin     //bge
                if($signed(op1) >= $signed(op2)) br=1;
                else br=0;
            end 
            5:begin     //bltu
                if(op1 < op2) br=1;
                else br=0;
            end 
            6:begin     //bgeu
                if(op1 >= op2) br=1;
                else br=0;
            end 
            default:begin
                br=0;
            end 
        endcase
    end
endmodule

NPC_SEL选择模块

这个模块的作用是选择下一个需要执行指令的地址。这个的选择取决于jal、jalr和br信号，皆高电平有效，Control模块对信号的处理决定了这三个信号在同一时间或都无效，或有且只有一个有效。对于jal和br信号，它们选择的都是同一个输入：pc_jal_br。因为它们的跳转地址都是pc+imm；这个值由ALU计算得出，所以输入端口pc_jal_br外接alu_res。jalr有效时，跳转地址是rd0+imm，同样由ALU计算得出。jalr比jal和branch指令多一个与操作。如果三个信号都无效，即无需跳转，所以程序按顺序执行下一条指令，选择器选择pc_add4作为pc_next的值。

module NPC_SEL (
    input jal,
    input jalr,
    input br,
    input [31:0] pc_add4,
    input [31:0] pc_jal_br,
    input [31:0] pc_jalr,
    output reg [31:0] pc_next
);
    always @(*) begin
        if(jal==1 || br==1)
            pc_next=pc_jal_br;
        else if(jalr==1)
            pc_next=pc_jalr;
        else pc_next=pc_add4;
    end
endmodule

算术逻辑单元ALU：代码如下，of为加减法运算的溢出标志信号，由于我们不需要考虑这种情况，故例化时将of信号悬空即可。
ALU的其他应用实例可参考另一篇文章：ALU的硬件编程实现及其应用

module ALU (
    input [31:0] a, b,     //两操作数（对于减运算，a是被减数）
    input [3:0] func,           //操作功能（加、减、与、或、异或等）
    output reg [31:0] y,       //运算结果（和、差 …）
    output reg of
);

wire [31:0] minus_b;
assign minus_b=~b+1;    //a-b=a+(-b)

always @(*) begin
    case(func)
        4'b0000:y=a+b;
        4'b0001:y=a-b;
        4'b0010:y=(a==b? 1:0);
        4'b0011:y=(a<b? 1:0);
        4'b0100:y=($signed (a) < $signed (b) ? 1:0) ;
        4'b0101:y=a&b;
        4'b0110:y=a|b;
        4'b0111:y=a^b;
        4'b1000:y=a>>b;
        4'b1001:y=a<<b;
        default:y=0;
    endcase
end
always @(*) begin
    if(func==0)begin        //加法运算溢出讨论
        if(a[31]==b[31] && a[31]^y[31])
            of=1;
        else of=0; 
    end
    else if(func==1) begin      //减法运算溢出讨论
        if(a[31]==minus_b[31] && a[31]^y[31])
            of=1;
        else of=0; 
    end
    else of=0;
end
endmodule

还有众多其他小模块，原理很简单，不一一赘述了，下面给出它们的实现：

module PC(
    input clk,
    input rst,
    input [31:0] pc_next,
    output reg [31:0] pc_cur
);
    always @(posedge clk or posedge rst) begin
        if(rst)
            pc_cur<=32'h2ffc;
        else
            pc_cur<=pc_next;
    end
endmodule

module RF             //三端口32 xWIDTH寄存器堆
(   
    input clk,                      //时钟（上升沿有效）
    input [4 : 0] ra0,               //读端口0地址
    output [31 : 0] rd0,      //读端口0数据
    input [4: 0] ra1,                //读端口1地址
    output [31 : 0] rd1,      //读端口1数据
    input [4:0] ra_dbg,
    output [31:0] rd_dbg,
    input [4 : 0] wa,                //写端口地址
    input we,                       //写使能，高电平有效
    input [31 : 0] wd         //写端口数据
);
    reg [31 : 0] Regfile[0 : 31];

    integer i;
    initial begin
        i = 0;
        while (i < 32) begin
        Regfile[i] = 32'b0;
        i = i + 1;
        end
        Regfile[2] = 32'h2ffc;
        Regfile[3] = 32'h1800;
    end

    assign  rd0 = Regfile[ra0],
            rd1 = Regfile[ra1],
            rd_dbg = Regfile[ra_dbg];
    always @ (posedge clk) begin
        Regfile[0]<=0;
        if (we==1 && wa!=0) Regfile[wa] <= wd;
    end
endmodule


module ADD(
    input [31:0] lhs,
    input [31:0] rhs,
    output [31:0] res
);
    assign res = lhs + rhs;
endmodule

module AND(
    input [31:0] lhs,
    input [31:0] rhs,
    output [31:0] res
);
    assign res = lhs & rhs;
endmodule

module MUX1 (
    input sel,
    input [31:0] src0,
    input [31:0] src1,
    output reg [31:0] res
);
    always @(*) begin
        if(sel)
            res=src1;
        else 
            res=src0;
    end
endmodule

module MUX2 (
    input [1:0] sel,
    input [31:0] src0,
    input [31:0] src1,
    input [31:0] src2,
    input [31:0] src3,
    output reg [31:0] res
);
    always @(*) begin
        case (sel)
            0:begin
                res=src0;
            end
            1:begin
                res=src1;
            end
            2: begin
                res=src2;
            end
            default: begin
                res=src3;
            end
        endcase
    end
endmodule
